# Glossary Format

The `GLOSSARY.md` file captures the canonical language for the teaching workspace. It's a living document — add terms as they're resolved.

## Structure

```markdown
# Glossary

**Term**:
A concise, opinionated definition of what this term means in this context.
_Avoid_: synonym1, synonym2
```

## Rules

- Add a term only when the user understands it — don't pre-populate
- Be opinionated: pick the best term and list alternatives under `_Avoid_`
- Keep definitions tight — one or two sentences max
- Revise entries as understanding deepens
