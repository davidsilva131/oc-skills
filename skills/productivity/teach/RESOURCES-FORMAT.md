# Resources Format

The `RESOURCES.md` file lists high-quality external resources.

## Structure

```markdown
# Resources

## Knowledge
- [Title](url) — one-line annotation. Why this resource, what it covers well.

## Wisdom
- [Title](url) — one-line annotation. Why this perspective matters.
```

## Rules

- High-trust only — vet every resource before listing
- Annotate every entry with why it's useful
- Prune ruthlessly — a long list is worse than no list
