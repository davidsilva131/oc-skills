# Decision Mapping

Turn a loose idea into a sequenced map of investigation tickets, then drive them to resolution one at a time.

## Decision map

A decision map is a markdown file with tickets keyed by dash-case slug. Each ticket has a type (Research, Prototype, Grilling) and dependencies.

## Ticket types

- **Research** — explore and report back. No code.
- **Prototype** — build a throwaway to answer a specific question.
- **Grilling** — interview the user to resolve ambiguity.

## Workflow

1. **Create the map**: elicit the idea, break it into tickets, sequence by dependency
2. **Work through**: resolve one ticket per session, never more
3. **Handoff**: every session ends with a handoff document containing a "Next steps" block
